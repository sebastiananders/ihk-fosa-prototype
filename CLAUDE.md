# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is a static HTML/CSS/JS prototype for **IHK FOSA** — a German foreign qualification recognition platform (Anerkennung ausländischer Berufsabschlüsse). It demonstrates a complete applicant journey and an admin view using the **KERN UX Design System** (`@kern-ux/native`), a German government open-source design system.

## Commands

```bash
# Install dependencies
npm install

# Build: copies KERN assets from node_modules into src/assets/kern/
npm run build
```

There are no test or lint scripts configured. To run locally, open `src/index.html` directly in a browser after running the build step.

Vercel is configured to run `npm run build` and serve from the `src/` directory (`vercel.json`).

## Architecture

**All pages are static HTML** with vanilla JS. There is no framework, no bundler, and no backend.

### Page Structure

| File | Role |
|------|------|
| `src/index.html` | Prototype overview — links to all demo pages |
| `src/login.html` | Login form; routes to admin or applicant based on email |
| `src/registrierung.html` | 4-step registration wizard with progress indicator |
| `src/status.html` | Applicant status tracker with 5-stage timeline |
| `src/admin-antraege.html` | Admin board: searchable, filterable application list |

### CSS Layer

| File | Purpose |
|------|---------|
| `src/theme.css` | IHK-FOSA brand tokens + KERN overrides. Contains the "Civic Digital" color philosophy and WCAG contrast documentation. Load this first. |
| `src/components.css` | Custom components not covered by KERN: bottom nav, timeline, registration steps, admin board, password toggle. |

KERN system CSS and JS are loaded from `src/assets/kern/` (generated by the build step, excluded from git).

### Design System (KERN)

The only npm dependency is `@kern-ux/native`. It provides typography (Fira Sans), color tokens, form elements, buttons, cards, alerts, and badges. The IHK theme overrides KERN's CSS custom properties in `theme.css`.

KERN documentation is available at `www.kern-ux.de` and `gitlab.opencode.de` (Claude Code has WebFetch access to these domains).

### JavaScript

All JS is inline `<script>` blocks in each HTML file. Key patterns:
- **login.html**: Client-side only; emails containing "admin" route to the admin board, others to status page.
- **registrierung.html**: Step wizard controlled by `currentStep` variable and `showStep()` function.
- **admin-antraege.html**: Live search and status filter over hardcoded sample data.

No JS modules, no build pipeline for scripts.

### Asset Path Dependency

`src/assets/kern/` is generated and git-ignored. **`npm run build` must be run before opening pages in a browser**, otherwise styles and fonts will be missing.
